30-Step Masterclass

Zero to Hero:
The Antigravity Epic

A completely unbroken, hand-holding masterclass. 30 detailed steps narrating exactly how to shatter the Ollama context ceiling and permanently fuse Gemma 4 to the Hermes Agent.

Scroll to begin the journey ↓

01

The Awakening

Every hero begins with realizing they are trapped. You’ve noticed Hermes hitting invisible walls when analyzing massive codebases.

Open your system terminal. We are about to bypass the 4K token limit imposed by standard APIs.

02

Installing Ollama

Ollama is the engine that will run our intelligence locally, entirely off-grid.

Download and install Ollama from ollama.com if you haven't already. Ensure the Ollama icon is running in your system tray.

03

Summoning Gemma 4

We need raw intelligence. We pull the uncensored, 12-Billion parameter Gemma 4 model directly to your machine.

Run this command to download the base Heretic GGUF model into Ollama's local cache.

Terminal / Configuration
ollama pull igorls/gemma-4-12B-it-heretic-GGUF:latest
04

Verifying the Download

A quick check ensures the model has descended correctly into your local architecture.

List your installed models to confirm it's ready for manipulation.

Terminal / Configuration
ollama list
05

Understanding the Bottleneck

Hermes talks to Ollama via an OpenAI endpoint. This endpoint strips context limits, aggressively capping generation to 4,096 tokens.

Understand that changing Hermes settings won't fix this. We must manipulate the model at the source.

06

The Modelfile Blueprint

To bypass the API, we create a genetic blueprint for a new model: a Modelfile.

Open your desktop folder, right-click, create a new text document, and name it 'Modelfile'. Remove the '.txt' extension.

07

Declaring the Base

Every blueprint needs a foundation. We tell the Modelfile to inherit everything from the Gemma model we just downloaded.

Open the Modelfile in Notepad and paste the FROM declaration.

Terminal / Configuration
FROM igorls/gemma-4-12B-it-heretic-GGUF:latest
08

Shattering the Context Wall

We force the absolute maximum context window. This allows the model to remember an entire book of code at once.

Add the num_ctx parameter to hardcode 65,536 tokens.

Terminal / Configuration
PARAMETER num_ctx 65536
09

Infinite Generation

Standard models artificially stop talking to save compute. We disable this entirely so it can generate massive scripts in one breath.

Add the num_predict parameter set to -1 (infinite).

Terminal / Configuration
PARAMETER num_predict -1
10

Hardware Saturation

We command Ollama to push as many neural layers as possible into your 12GB VRAM before safely spilling into your 64GB System RAM.

Add the num_gpu parameter set to 999 to guarantee maximum GPU offloading.

Terminal / Configuration
PARAMETER num_gpu 999
11

Reviewing the Blueprint

Before we compile, we must ensure our genetic modifications are perfect.

Your complete Modelfile should look exactly like this. Save and close Notepad.

Terminal / Configuration
FROM igorls/gemma-4-12B-it-heretic-GGUF:latest
PARAMETER num_ctx 65536
PARAMETER num_predict -1
PARAMETER num_gpu 999
12

Compiling the Alias

We fuse the blueprint with the base model, giving birth to our limitless variant.

Open your terminal in the same folder as the Modelfile and run the create command to build 'hermes-gemma-4-12b'.

Terminal / Configuration
ollama create hermes-gemma-4-12b -f ./Modelfile
13

Watching the Build

Ollama reads your constraints, recalculates the layers, and writes the new manifest to disk.

Wait a few seconds for the terminal to say 'success'. It doesn't redownload the model, it just creates a lightweight alias pointer.

14

Validating the Unbottlenecked Model

We confirm our new alias is alive and recognized by the system.

Run the list command again to see your newly minted creation.

Terminal / Configuration
ollama list | findstr hermes-gemma
15

Opening Hermes

Now that the engine is built, we turn our attention to the vehicle: The Hermes Agent.

Navigate to the Hermes application data directory to locate the core configuration.

Terminal / Configuration
cd %LOCALAPPDATA%\hermes
16

Locating config.yaml

The config.yaml file is the brain stem of the Hermes UI and CLI.

Open config.yaml in any text editor.

17

Mapping the Primary Model

We must explicitly tell Hermes to abandon its default API model and point to our unbottlenecked local alias.

Find the 'model:' section and update the 'default' key.

Terminal / Configuration
model:
  default: hermes-gemma-4-12b
  provider: ollama
18

Setting the Local API Path

Hermes still communicates via API, but now it points locally.

Ensure the base_url points to your local Ollama port.

Terminal / Configuration
  base_url: http://localhost:11434/v1
19

Injecting Dummy Keys

Ollama doesn't need an API key, but the OpenAI endpoint format expects one to not crash.

Add a dummy API key under the base_url.

Terminal / Configuration
  api_key: sk-dummy-key-not-used
20

Defining Context Lengths

Although our Modelfile forces 65k, Hermes needs to know it's allowed to send that much text in its internal UI token counter.

Update the context parameters to match our Modelfile exactly.

Terminal / Configuration
  ollama_num_ctx: 65536
  context_length: 65536
21

Updating the Delegation Model

This is where most novices fail. Hermes spawns invisible background agents to do research. If you don't update them, they crash.

Scroll down to the 'delegation:' block.

22

Syncing the Sub-Agents

We force the background researchers to use the exact same unbottlenecked alias as the main chat window.

Update the delegation model string.

Terminal / Configuration
delegation:
  model: hermes-gemma-4-12b
  provider: ollama
23

Configuring the File Editor Tool

Hermes uses a specific sub-agent to edit massive files. It needs our new model to handle thousands of lines of diffs.

Scroll to the 'tool_agent:' block (if it exists) and update its model.

Terminal / Configuration
tool_agent:
  model: hermes-gemma-4-12b
24

Saving the Configuration

The genetic modifications are complete. The brain stem is rewritten.

Save config.yaml and close your text editor.

25

Booting the Hermes App

We awaken the UI. It will now read the configuration and lock onto our local Ollama instance.

Double-click the Hermes Windows App icon on your desktop.

26

Verifying the UI Connection

The bottom right corner of Hermes should now proudly display your custom alias.

Look at the model selector in the UI. It should say 'hermes-gemma-4-12b (ollama)'.

27

The Stress Test

True heroes test their limits. We throw a massive query at the model to ensure the 65k context holds.

In the Hermes chat, paste a massive block of code or an entire essay and ask it to summarize.

28

Monitoring Hardware (Optional)

Watch your machine seamlessly handle the load. Your 12GB VRAM will max out, and the rest will spill flawlessly to your 64GB System RAM.

Open Task Manager (Ctrl+Shift+Esc) and watch the Performance tab during generation.

29

Testing the Sub-Agents

We must verify that Hermes can delegate tasks successfully without crashing.

Ask Hermes: 'Please spawn a sub-agent to write a python script for me.' Watch the delegation succeed.

30

Ascension Complete

You have transcended. You are no longer bound by artificial API bottlenecks. You possess a limitless, uncensored local supercomputer.

Enjoy the power of the true Antigravity stack. There are no more limits.
