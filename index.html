The Proper Fix

A realistic, sane guide to configuring context length in Ollama and Hermes Agent without crashing your system or falling for fake hype.

🧠 The Reality Check

Myth vs. Reality

Before we start, it's important to know that a model can't magically use more context than it was designed for. Each model has a built-in "architectural maximum," like Llama 3.1's 128K tokens. Setting a num_ctx value higher than this ceiling will just be ignored, and the model will silently work at its true limit. Furthermore, infinite generation (num_predict -1) is dangerous advice that can lead to runaway compute.

🛠️ Step-by-Step Setup

Here’s how to properly set up Hermes Agent to work with local models using Ollama, focusing on real-world configuration and sane context management.

Step 1: Set Your Default Context Length in Ollama

For a reliable baseline, the official Ollama documentation recommends setting a global default context length using the OLLAMA_CONTEXT_LENGTH environment variable.

  • On Windows: Quit Ollama (from the system tray), set a new environment variable named OLLAMA_CONTEXT_LENGTH to your desired value (e.g., 32768 for a 32K context), and then restart Ollama.
  • On macOS/Linux: Use launchctl or systemctl to set the environment variable before starting the Ollama service.

Step 2: Create a Custom Modelfile

If you need a consistent context for a specific task without messing with environment variables, you can bake it directly into a model alias.

Terminal (bash)
# Create a new Modelfile
cat > Modelfile << 'EOF'
FROM llama3.1:8b  # Or pull your preferred model like "qwen2.5-coder:7b"
PARAMETER num_ctx 32768 # Set your desired, sane context window here
EOF

# Build the new model alias
ollama create my-longcontext-model -f ./Modelfile

Step 3: Configure Hermes Agent Correctly

Hermes Agent needs to be explicitly told to use your local Ollama instance and respect the exact context limits you established. Open your ~/.hermes/config.yaml and define a custom provider.

config.yaml
custom_providers:
  - name: "my_local_ollama"
    base_url: "http://localhost:11434/v1"
    api_key: "ollama" # Can be any non-empty string
    models:
      - id: "my-longcontext-model"
        context_length: 32768 # Must match the num_ctx set in Ollama

Set this as your active model via CLI: hermes model set my_local_ollama:my-longcontext-model.

🎯 Hardware Limitations: A Real-World Roadmap

Getting the technical steps right is only half the battle. The KV cache (the memory that stores the conversation history) grows alongside your context window. Setting a context length too high will cause Ollama to exceed your VRAM, forcing it to offload to slower system RAM or crash entirely.

Total VRAM Recommended Context Reasoning
6–8 GB 4,096 - 8,192 tokens Prioritizes stable model loading and generation. Limited VRAM can't handle larger contexts without crashing.
10–12 GB 16,384 - 32,768 tokens Balances decent context length with model weights. Good for many coding and analysis tasks.
16-24+ GB 32,768 - 64,000+ tokens Enables handling of very large documents or extensive codebases. VRAM requirement grows quickly.

💎 Troubleshooting

  • Out of Memory / Crashing: Check the Ollama logs for errors related to out-of-memory or context truncation. Run OLLAMA_DEBUG=1 ollama serve to see detailed logs.
  • Slow Performance: Use ollama ps to monitor your model's memory usage. It will show you if the model is fully loaded on your GPU or if it has spilled over to your CPU (which drastically slows down generation).
  • Persistent Issues: Consider using a smaller model (e.g., moving from a 12B to an 8B/7B) or a highly quantized version to free up more VRAM for the context cache.
