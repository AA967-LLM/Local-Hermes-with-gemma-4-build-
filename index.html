A comprehensive, step-by-step guide to removing artificial bottlenecks from Ollama, unlocking the full 65,536 context window for the uncensored Gemma 4 12B model, and tying it natively to the Hermes Agent on Windows.

1. The OpenAI Compatibility Bottleneck

By default, Hermes interfaces with local Ollama instances via the OpenAI-compatible /v1/chat/completions endpoint. While highly compatible, this endpoint has a fatal flaw: it strips out context window parameter overrides like num_ctx.

The Result: Even if you tell Hermes to use 65,536 tokens, the API defaults your Ollama model to a suffocating 4,096 tokens. This causes massive contextual generation tasks to truncate mid-sentence.

To fix this, we do not modify the Hermes code. Instead, we compile a native Ollama Modelfile that physically forces the model to load with maximum overhead.

2. Creating the Unbottlenecked Alias

Create a file named Modelfile anywhere on your system with the following parameters. This builds an alias based on the uncensored Gemma 4 model, explicitly maximizing the context, disabling generation limits, and forcing maximum layers to your 12GB VRAM.

FROM igorls/gemma-4-12B-it-heretic-GGUF:latest

# Force 65k context window natively
PARAMETER num_ctx 65536

# -1 disables artificial generation stopping limits
PARAMETER num_predict -1

# Force maximum layers to GPU (overflows safely to System RAM)
PARAMETER num_gpu 999

Once saved, open your terminal and build the model alias:

ollama create hermes-gemma-4-12b -f ./Modelfile

3. Tying Hermes to the Hip

Now that the unbottlenecked model exists, we must instruct both the Hermes CLI and the Windows Desktop App to use it. Open your Hermes configuration file located at %LOCALAPPDATA%\hermes\config.yaml.

Find the primary model section and update it:

model:
  default: hermes-gemma-4-12b
  provider: ollama
  base_url: http://localhost:11434/v1
  api_key: sk-a918f76395b0429780000a6e70901d88
  ollama_num_ctx: 65536
  context_length: 65536
  max_tokens: 8192

Crucially, ensure you also update the background sub-agent delegation so that Hermes doesn't fall back to a deleted model when spawning workers:

delegation:
  model: hermes-gemma-4-12b
  provider: ollama
  base_url: http://localhost:11434/v1

4. Resulting Performance

With 64GB of System RAM and 12GB of VRAM, this configuration allows the Gemma-4 model's KV Cache to comfortably spill over into system memory without crashing.
