0 to Hero Guide

The Unbottlenecked
Local AI Stack

You shouldn't have to waste your entire day debugging context limits. Here is the definitive, total-novice guide to flawlessly linking the Hermes App, Ollama, and Gemma 4.

01

The Setup: Getting the Engine

Before we can remove the bottlenecks, you need the base model. We are using Gemma-4-12B—an incredibly smart, uncensored model that fits perfectly into 12GB VRAM while overflowing safely into your 64GB of System RAM.

Terminal / Command Prompt
# Pull the base model down to your machine
ollama run igorls/gemma-4-12B-it-heretic-GGUF:latest

Once it finishes downloading, you can type "/bye" to exit. We don't want to run it like this—we want to superpower it first.

02

The Problem: The Silent Chokehold

Hermes communicates with Ollama using an "API endpoint". Unfortunately, this endpoint has a fatal flaw: it silently ignores context limits and crushes your model down to a tiny 4,096 tokens. If you paste a large document, the AI will abruptly cut off.

The Fix: We bypass the API completely. By creating a physical "Modelfile" (a configuration blueprint), we hardcode the maximum limits directly into the model's DNA.
03

The Solution: Building the Alias

Create a blank text file on your desktop and name it exactly Modelfile (no .txt extension). Paste the following code into it exactly as written:

Modelfile
FROM igorls/gemma-4-12B-it-heretic-GGUF:latest

# 1. Force the absolute maximum 65,536 context window
PARAMETER num_ctx 65536

# 2. Set to -1 to disable artificial generation cut-offs
PARAMETER num_predict -1

# 3. Force all layers to GPU to maximize your 12GB VRAM
PARAMETER num_gpu 999

Now, open your terminal, navigate to your desktop, and tell Ollama to compile this blueprint into a brand new, unbottlenecked model named hermes-gemma-4-12b:

Terminal / Command Prompt
ollama create hermes-gemma-4-12b -f ./Modelfile
04

The Fusion: Tying it to Hermes

You now have a limitless AI engine. Finally, we must plug it into the Hermes Windows App and CLI so they are "tied at the hip".

Open your Hermes configuration file located at C:\Users\YourName\AppData\Local\hermes\config.yaml.

Find the primary model section and update it to our new alias:

config.yaml (Section 1)
model:
  default: hermes-gemma-4-12b
  provider: ollama
  base_url: http://localhost:11434/v1
  ollama_num_ctx: 65536
  context_length: 65536

Crucially, scroll down and find the delegation section. Hermes spawns "sub-agents" to do background research. If you don't update this, the background agents will crash.

config.yaml (Section 2)
delegation:
  model: hermes-gemma-4-12b
  provider: ollama
  base_url: http://localhost:11434/v1
05

You are now a Hero.

Restart Hermes. You now possess a fully local, 100% uncensored AI stack capable of injesting tens of thousands of lines of code without hallucinating or abruptly stopping.

Welcome to the true Antigravity experience.
